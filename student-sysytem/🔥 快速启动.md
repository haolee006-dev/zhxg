## 🚀 三种启动方式

### 方式一：双击启动脚本（最简单！）⭐⭐⭐

1. **启动后端**（如果没运行）：
   - 双击打开：`student-system-backend\启动后端.bat`

2. **启动前端**：
   - 双击打开：`student-system-frontend\启动前端.bat`

3. **访问系统**：
   - 浏览器打开：http://localhost:3000

---

### 方式二：手动使用完整路径执行

如果双击脚本也提示"选择应用"，使用完整路径：

#### 启动前端：
```cmd
# 打开命令提示符（不是 PowerShell！）
# 按 Win+R，输入 cmd，回车

cd D:\lee\zhxg\student-sysytem\student-system-frontend

# 使用完整路径执行 npm
"C:\Program Files\nodejs\npm.cmd" install --registry=https://registry.npmmirror.com
"C:\Program Files\nodejs\npm.cmd" run serve
```

---

### 方式三：修复文件关联问题

如果所有 .cmd/.bat 文件都提示"选择应用"：

1. **右键点击**任意 `.bat` 或 `.cmd` 文件
2. 选择 **"打开方式"** → **"选择其他应用"**
3. 向下滚动，点击 **"更多应用"**
4. 再向下滚动，点击 **"在这台电脑上查找其他应用"**
5. 浏览到：`C:\Windows\System32\cmd.exe`
6. 选择 `cmd.exe`，点击 **"打开"**
7. 勾选 **"始终使用此应用打开 .bat/.cmd 文件"**
8. 点击 **"确定"**

修复后，就可以正常双击启动脚本了！

## ❓ 常见问题

### Q: 双击 .bat 文件还是提示"选择应用"？
A: 见上方"方式三：修复文件关联问题"

### Q: npm 命令无法识别？
A: 使用完整路径：`"C:\Program Files\nodejs\npm.cmd"`

### Q: 后端启动失败？
A: 检查：
   - JDK 21 是否安装
   - MySQL 是否正在运行
   - 数据库 student_system 是否已创建

### Q: 前端启动失败？
A: 先确保依赖已安装：`npm install`
